    Hugo搭建博客

    “Hugo是由Go语言实现的静态网站生成器。简单、易用、高效、易扩展、快速部署。”

    • Hugo Github传送门

    • Hugo中文文档

    一、Hugo安装

    安装包下载地址：https://github.com/gohugoio/hugo/releases

    1、Windows安装

    • 从下载地址中下载 hugo_extended_x.x.x_Windows-xbit.zip
    • 下载成功后解压，并在环境变量 path 中添加路径。

    2、Linux(Ubuntu)安装

    • 从下载地址中下载 hugo_extended_x.x.x_Linux-xbit.tar.gz

    • 将压缩包移到自定义目录下进行解压

      > mv hugo_extended_0.58.3_Linux-64bit.tar.gz Hugo/
> tar -zxvf hugo_extended_0.58.3_Linux-64bit.tar.gz

    • 将文件夹下的执行文件hugo软链接到/usr/local/bin

      > sudo ln -s $(你解压的绝对路径)/hugo /usr/local/bin/hugo

    3、macOS安装

    > brew install hugo

    4、验证是否配置成功

    个人推荐windows平台使用Git Bash，其中可以直接执行Linux命令

    > hugo version
Hugo Static Site Generator v0.58.3/extended windows/amd64 BuildDate: unknown

    二、创建博客

    1. 创建项目(注：MyBlog为项目名可自定义)

    > hugo new site MyBlog

Congratulations! Your new Hugo site is created in /xxx/xxx/myblog.

Just a few more steps and you're ready to go:

1. Download a theme into the same-named folder.
   Choose a theme from https://themes.gohugo.io/ or
   create your own with the "hugo new theme <THEMENAME>" command.
2. Perhaps you want to add some content. You can add single files
   with "hugo new <SECTIONNAME>/<FILENAME>.<FORMAT>".
3. Start the built-in live server via "hugo server".

Visit https://gohugo.io/ for quickstart guide and full documentation.

    2.添加版本管理

    • 在远程仓库创建 MyBlog 私有仓库

    • 进入项目根路径

      > cd MyBlog
>git init
>git add .
>git commit -m "*****"
>git remote add origin https://*********
>git push -u origin master

    3.下载主题并添加版本控制

    • 主题地址，选择自己钟意的一项，每个主题都会有使用说明和例子 下载主题，这里我使用 m10c 进行演示

      注意: 由于我们已经将MyBlog添加了版本控制，如果将主题clone下来也会存在版本控制，所以如果我们执行 git add . 时将会出现 git 无法获取两个仓库的嵌套关系的异常。下面提供两种方案，请理解后再进行操作

      方案一、将主题做为submodule（相关操作请看文章末尾）
      > git submodule add https://github.com/vaga/hugo-theme-m10c.git themes/m10c

      添加为 submodule后，如果 m10c 的作者更新了项目，你就只需在 m10c 目录下拉取更新就行了。但是如果自己 修改了主题里的内容的话，就无法提交，所以我们可以先 fork 项目到自己的仓库，然后再做添加为 submodule

      方案二、删除m10c目录下的.git文件夹，移除m10c版本控制
      >git clone https://github.com/vaga/hugo-theme-m10c.git themes/m10c

      clone之后进入theme/m10c删除.git文件夹,移除了主题先前的版本控制,之后就可以将主题添加到 MyBlog 仓库的版本控制

    • 设置主题，在文件中添加 theme = "m10c"

      >vim config.toml

      到这里仅完成了主题的基本配置，如果需要进行更多的配置可以参考 themes/m10c/exampleSite/config.toml 中是如何配置的，关于主题的个性化修改需要自己探索

    4.创建文章

    • 确认项目根目录MyBlog/下是否有content文件夹，创建的文章必须要在 content 下面

      > mkdir content

    • 创建第一篇文章放到 content/post 目录

      > hugo new post/first.md

      first.md 内容

      ---
title: "First"
date: 2019-10-16T16:29:06+08:00
draft: true
---

      打开first.md 可以看到已经有了三个文件头，可以在 archetypes/default.md 中可修改默认的配置

    5.本地运行调试

    • 查看草稿文章 (draft = true)

      > hugo server -D 或者 hugo server --buildDrafts

Web Server is available at http://localhost:xxxx/ (bind address 127.0.0.1)

    • 查看正式文章 (draft = false)

      > hugo server

Web Server is available at http://localhost:xxxx/ (bind address 127.0.0.1)

    在浏览器输入地址,即可通过编辑MD文件并在浏览器中实时预览内容

    6.生成静态网页资源

    • 将需要发布的文章内的属性draft改为false

      > hugo

      运行成功后，根目录下多出public文件夹出来，文件夹中的内容就是 Hugo 生成的静态网页

    三.博客部署到Gitee或者GitHub

    我们博客主要就是通过静态网页托管服务来实现的，所以我们要需要将Hugo生成的静态网页放在提供服务的平台。 由于Github我已经部署了博客，所以这里以Gitee为例，虽然具体有些差别但是官方文档里都有说明。

    1.创建Pages服务

    Gitee Pages文档

    Github Pages文档

    • 首先在 Gitee 上创建 的公开项目, 由于域名并不区分大小写，所以这里全小写形式就行了
    • 「仓库主页」->「服务」->「Gitee Pages」打开 Gitee Pages 服务， 并强制使用HTTPS，服务开启后Gitee会提供一个Url，这个Url就是博客的地址了， 同时项目中 config.toml 的 baseURL 要设置成提供的Url

    2. 上传到远程仓库

    • 进入MyBlog/public目录

      >git init
>git add .
>git commit -m "*****"
>git remote add origin https://*********
>git push -u origin master

      push 项目后就可以通过 Gitee 提供的 Url 进行访问

    3.将博客托管的项目做为submodule加入MyBlog

    >cd MyBlog/
>git submodule add https://gitee.com/kittoZ/kittoZ.git  public/

    由于Hugo生成的静态页面会在public/文件夹下，所以我们将子模块存储在public/，如果我们更新了文章后 只需要提交该子模块，就能更新博客了

    知识点：Git子模块管理

    1.添加子模块

    > git submodule add <url> <path>
    • url 为子模块的路径(本文为博客git地址)， path 为子模块存储的目录路径，执行完需重新提交项目

    2.克隆壳工程

    • 重新克隆完壳工程后，子模块并不会一同出现，需要进行初始化

      >git submodule init
>git submodule update
或者：
>git submodule update --init --recursive

    • 子模块更新，当子模块有了新的提交，如何更新本地项目呢？

      • 首先进入子模块,拉取更新

        >git pull

      • 返回项目根目录,重新提交项目

    3.删除子模块

    处于项目根目录，全部执行完记得重新提交项目

    • 将子模块从版本控制中移除

      >git rm --cached 子模块名称

    • 删除子模块目录

      >rm -rf 子模块目录

    • 删除.gitmodules需要删除子模块信息

      >vim .gitmodules

    • 删除文件中的子模块信息

      >vim .git/config

    • 删除模块下的子模块目录

      >rm -rf .git/module/子模块名称
